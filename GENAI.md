# GenAI Usage — Task Manager

## Tool Used
**Claude Code** (claude-sonnet-4-6) via Anthropic CLI — an agentic AI coding assistant that can read/write files and run commands directly in the terminal.

## How AI Was Used

### 1. Project Scaffolding
- All `dotnet new` commands, NuGet package additions, and project reference wiring were generated and executed by the AI agent.
- The complete folder structure (Domain/Application/Infrastructure/Api/Auth and their test counterparts) was planned and created in a single session.

### 2. Backend Code Generation
The AI generated the full backend implementation:
- **Domain layer**: `TaskItem`, `User` entities with factory methods; `Email` and `TaskStatusValue` value objects; domain exceptions; repository interfaces
- **Application layer**: Service classes (`TaskService`, `AuthService`), DTOs (records), FluentValidation validators, `IPasswordHasher` abstraction
- **Infrastructure layer**: Npgsql raw SQL repositories (no EF/Dapper/Mediator), `JwtService`, `PasswordHasher`, `DbConnectionFactory`, `DatabaseMigrator` with idempotent DDL and seed data
- **API/Auth layers**: Controllers, JWT Bearer configuration, exception middleware, Swagger setup, `Program.cs` DI wiring

### 3. Bug Fixing
- The AI identified and resolved a `TaskStatus` namespace collision with `System.Threading.Tasks.TaskStatus` by introducing a `using DomainTaskStatus = ...` alias across affected files.
- Missing NuGet package references (`Microsoft.Extensions.DependencyInjection.Abstractions`, `Microsoft.Extensions.Configuration.Abstractions`, `System.IdentityModel.Tokens.Jwt`) were detected from build errors and added automatically.

### 4. Test Generation
All 37 unit tests were generated by AI:
- Domain: `TaskItemTests` (8 tests), `EmailTests` (5 tests)
- Application: `TaskServiceTests` (6 tests), `AuthServiceTests` (5 tests)
- Infrastructure: `TaskRepositoryTests` (4 tests — structural/contract)
- API: `TasksControllerTests` (2 tests — integration with `WebApplicationFactory`)

### 5. Frontend Scaffolding
- Angular 17 standalone project scaffolded with `@angular/cli`
- All components (`LoginComponent`, `RegisterComponent`, `TaskListComponent`, `TaskCardComponent`, `TaskFormComponent`) generated with inline templates and styles
- Auth interceptor (functional `HttpInterceptorFn`), `AuthGuard` (functional `CanActivateFn`), and services generated in full
- Reactive Forms with validation on all forms

### 6. Infrastructure Files
- `docker-compose.yml` with Postgres + Auth + API + UI services
- Multi-stage `Dockerfile` for each service
- Nginx config for Angular SPA routing

## What Was NOT AI-Generated
- Architecture decisions (Clean Architecture, DDD patterns, Npgsql raw requirement) — these were defined by the developer in the initial specification.
- Project requirements and acceptance criteria in `USER_STORY.md` — reviewed and validated by the developer.
- The decision to use `IPasswordHasher` abstraction in Application (instead of BCrypt directly) was a Clean Architecture correction made during the session.

## Validation
All generated code was compiled and tested:
- `dotnet build` — zero errors
- `dotnet test` — 37/37 tests passing
- Build output reviewed for warnings (nullable reference annotations intentionally not enforced on private constructors of domain entities using the factory pattern)
